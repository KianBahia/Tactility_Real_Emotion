🚀 HOW TO BUILD FOR PRODUCTION (3 STEPS)
================================================================================

STEP 1: Open Xcode
──────────────────────────────────────────────────────────────────────────────
   open ios/RealEmotion.xcworkspace


STEP 2: Switch to Release Configuration
──────────────────────────────────────────────────────────────────────────────
   In Xcode:
   1. Product → Scheme → Edit Scheme...
   2. Click "Run" in left sidebar
   3. Build Configuration: Change from "Debug" to "Release"
   4. Close the scheme editor


STEP 3: Build and Run
──────────────────────────────────────────────────────────────────────────────
   Press: Cmd + R

   The app will:
   ✅ Use embedded JavaScript bundle
   ✅ Work WITHOUT Metro bundler
   ✅ Work WITHOUT network connection
   ✅ Run completely standalone!


================================================================================
📱 TESTING ON DEVICE
================================================================================

1. Connect your iPhone via USB
2. Select your iPhone in Xcode device selector
3. Make sure scheme is set to "Release"
4. Press Cmd + R

To verify it's standalone:
• Turn OFF WiFi on your iPhone
• Turn OFF Cellular Data
• App should still work perfectly!


================================================================================
📦 FOR APP STORE SUBMISSION
================================================================================

1. Open Xcode:
   open ios/RealEmotion.xcworkspace

2. Select target:
   "Any iOS Device (arm64)"

3. Archive:
   Product → Archive

4. Wait for build (2-5 minutes)

5. When Organizer opens:
   • Click "Distribute App"
   • Select "App Store Connect"
   • Follow upload wizard

================================================================================
🐛 TROUBLESHOOTING
================================================================================

"App won't launch" or "Black screen":
   → Make sure you're using Release configuration
   → Try: Product → Clean Build Folder (Cmd + Shift + K)
   → Then rebuild

"Could not connect to development server":
   → You're in Debug mode! Switch to Release:
   → Product → Scheme → Edit Scheme → Release

"main.jsbundle not found":
   → Run: ./build-production.sh
   → Then rebuild in Xcode


================================================================================
📊 DEBUG vs RELEASE
================================================================================

DEBUG (Development):
   • Uses Metro bundler (network required)
   • Fast Refresh enabled
   • Dev menu available (shake device)
   • Slower startup
   • Larger file size

RELEASE (Production):
   • Embedded bundle (NO network needed)
   • Optimized & minified
   • No dev menu
   • Faster startup
   • Smaller file size
   • Ready for App Store
